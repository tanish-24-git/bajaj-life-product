# Bajaj Life Insurance Product Data Extraction

## Overview
This repository contains structured JSON data extracted from various Bajaj Life Insurance product pages and brochures. The data is organized into categories and follows a strict schema designed for RAG (Retrieval-Augmented Generation) applications.

## Total Products Extracted: 36

### Directory Structure
- **`term/`** (18 Products): Contains Term Insurance plans (e.g., various coverage amounts, senior citizens, women, specific plans like eTouch II, iSecure II).
- **`ulip/`** (10 Products): Contains Unit Linked Insurance Plans (e.g., Goal Assure, Future Wealth Gain, Invest Protect Goal).
- **`root`** (8 Products): Contains various Savings, Pension, and Retirement plans (e.g., ACE, Goal Suraksha, Saral Pension).

## JSON Data Format

Each product is saved as a single JSON file containing detailed information parsed into specific sections. Below is the schema description:

### Root Object Fields

| Field | Type | Description |
| :--- | :--- | :--- |
| `product_metadata` | Object | High-level details about the product. |
| `product_summary` | Object | A brief overview of the plan's objective and key highlights. |
| `plan_variants` | Array | List of available variants or options within the plan. |

| `retrieval_chunks` | Array | Segmented text chunks optimized for vector search/RAG. |

### Field Breakdown

#### 1. `product_metadata`
Contains identification details:
- `product_name`: Official name of the product.
- `category`: Type of plan (e.g., Term, ULIP, Savings).
- `uin`: Unique Identification Number (if available).
- `source_document`: Origin of the data (webpage or brochure PDF).

#### 2. `product_summary`
- `objective`: The primary goal of the plan (e.g., "Wealth Creation", "Protection").
- `key_highlights`: Bullet points of the most attractive features.
- `lock_in_rule`: (For ULIPs) Mention of the 5-year lock-in period.

#### 3. `core_benefits`
An array of objects, each describing a specific benefit:
- `benefit_name`: E.g., "Death Benefit", "Maturity Benefit".
- `details`: Comprehensive description of how the benefit works.
- `guaranteed_floor`: Any minimum guarantee (e.g., "105% of premiums paid").
- `payout_type`: How it is paid (Lump sum, Installments).

#### 4. `eligibility_and_parameters`
- `entry_age`: Min/Max age rules.
- `maturity_age`: Min/Max age at maturity.
- `policy_term_options`: Available duration of the policy.
- `premium_payment_term`: options (Regular, Limited, Single).
- `minimum_premium`: Thresholds for investment.

#### 5. `retrieval_chunks`
This section is specifically engineered for RAG. It contains small, self-contained text blocks tagged with relevant keywords.
- `chunk_id`: Unique ID for the chunk.
- `topic`: The specific topic covered (e.g., "Loyalty Additions").
- `text`: The actual text content to be embedded.
- `tags`: Keywords for filtering/hybrid search.
